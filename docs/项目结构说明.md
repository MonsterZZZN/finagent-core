# finagent-core 项目结构说明

> 这是项目的"地图"——每个目录和文件的作用都在这里。
> 随着开发推进，本文档会持续更新（标注 ✅ 已实现 / ⬜ 待实现）。

---

## 一、顶层结构

```
finagent-core/
├── README.md               ✅ 项目总览
├── .gitignore              ✅ git 忽略规则（venv/.env/缓存等）
├── requirements.txt        ✅ Python 依赖清单
├── .env.example            ✅ 环境变量模板（复制为 .env 填真实值）
├── .env                    ⬜ 真实环境变量（不进 git，你自己创建）
├── start_web.py            ⬜ Web 服务启动入口（步骤 8）
├── langgraph.json          ⬜ LangGraph 部署配置（步骤 7）
├── src/                    源代码目录
└── docs/                   文档目录
```

---

## 二、src/ 源代码目录

源代码按「职责」分成 4 个一级目录，一看就知道东西在哪：

| 目录 | 职责 | 一句话 |
|---|---|---|
| `src/agent/` | Agent 核心 | "大脑"：主 Agent、配置、中间件、子 Agent、工具 |
| `src/mcp_server/` | 数据工具服务 | "手脚"：通过 MCP 提供行情/财报/新闻等数据查询 |
| `src/skills/` | 技能包 | "操作手册"：风险分析/市场监控/报告生成的方法论+脚本 |
| `src/api_view/` | Web 接口 | "门面"：对外的 HTTP/SSE 接口 |

---

## 三、src/agent/ —— Agent 核心（最重要）

```
src/agent/
├── __init__.py
├── config.py               ⬜ 配置中心：模型/数据库/路径/常量（步骤2）
├── schema.py               ⬜ 数据结构：FinancialContext 等（步骤2）
├── main_agent.py           ⬜ 主 Agent 入口：create_deep_agent 组装（步骤7）
├── env_utils.py            ⬜ 环境变量读取工具（步骤2）
│
├── backends/               ⬜ 沙箱后端（隔离代码执行，步骤7）
│
├── memory/                 记忆与人格
│   ├── AGENTS.md           ⬜ 助手行为准则（主 Agent 的"说明书"，步骤3）
│   └── prompts.py          ⬜ 主 Agent system_prompt（步骤3）
│
├── middlewares/            中间件（在 Agent 运行前后做处理）
│   ├── context_injection.py    ⬜ 注入用户身份/持仓上下文（步骤7）
│   ├── compliance.py           ⬜ 合规检查（禁词扫描+风险提示，步骤10）
│   └── ...                      ⬜ 其他（技能同步/记忆更新/限流）
│
├── subagents/              子 Agent
│   ├── loader.py           ⬜ YAML 配置加载器（步骤6）
│   └── configs/            子 Agent 配置（每个 .yaml 一个子 Agent）
│       ├── risk_analyst.yaml      ⬜ 风险分析子 Agent（步骤6）
│       ├── market_pulse.yaml      ⬜ 市场监控子 Agent（步骤9后）
│       ├── report_writer.yaml     ⬜ 报告生成子 Agent（步骤9后）
│       └── research_proxy.yaml    ⬜ 研究代理子 Agent（步骤9）
│
└── tools/                  主 Agent 直接用的工具
    ├── mcp_client.py       ⬜ 连接 MCP Server 加载工具（步骤4）
    ├── research_proxy.py   ⬜ 调用 finagent-research（步骤9）
    ├── kb_query.py         ⬜ 调用 finagent-kb（步骤9）
    └── ...                 ⬜ 图表/搜索/技能管理等
```

**这层的核心理解**：
- `config.py` / `schema.py` 是地基，所有模块都依赖它
- `main_agent.py` 是"总装车间"，把模型+工具+子Agent+中间件组装成一个可运行的 Agent
- `subagents/configs/*.yaml` 是声明式配置——加一个新子 Agent 只要加一个 YAML，不改代码
- 子 Agent 之间不互相调用，全部由主 Agent 调度

## 四、src/mcp_server/ —— 数据工具服务

```
src/mcp_server/
├── __init__.py
├── server_main.py          ⬜ MCP Server 启动 + 注册所有工具组（步骤4）
├── server_config.py        ⬜ 数据 API 配置（步骤4）
├── http_base.py            ⬜ 生命周期管理（数据源客户端连接池，步骤4）
└── tools/                  各数据工具组（按业务分组）
    ├── market_data_tools.py    ⬜ 行情（实时/K线/指数/大盘，步骤4）
    ├── portfolio_tools.py      ⬜ 持仓管理
    ├── news_tools.py           ⬜ 新闻舆情
    ├── announcement_tools.py   ⬜ 公告/龙虎榜
    ├── earnings_tools.py       ⬜ 财报
    ├── futures_tools.py        ⬜ 期货
    └── macro_tools.py          ⬜ 宏观
```

**这层的核心理解**：
- MCP Server 是一个**独立进程**，专门提供"取数据"的工具
- 每个工具组是一个文件，工具命名 `{组名}_{动作}`（如 `market_data_quote`）
- Agent 通过 MCP 协议调用这些工具——好处是工具和 Agent 解耦，工具可以独立升级
- 数据来源：AKShare（免费）

## 五、src/skills/ —— 技能包（方法论 + 脚本）

```
src/skills/
├── main/                   通用技能
│   └── skill-management/   ⬜ 技能管理（保留 ERP 原项目的）
└── finance/                金融技能
    ├── risk-analysis/      风险分析（步骤5）
    │   ├── SKILL.md            ⬜ 操作手册（流程/模板）
    │   ├── scripts/           ⬜ 计算脚本（VaR/保证金/相关性/压测）
    │   ├── data/              ⬜ 配置（阈值/情景库/保证金规则）
    │   └── reference/         ⬜ 参考资料（方法论/文风）
    ├── market-monitor/     ⬜ 市场监控
    ├── report-generation/  ⬜ 报告生成
    └── data-query/         ⬜ 数据查询（NL2Query）
```

**这层的核心理解**（重要）：
- Skill = 「操作手册（SKILL.md）+ 可执行脚本（scripts/）+ 配置数据（data/）+ 参考资料（reference/）」
- **计算密集的逻辑（如 VaR 计算）放在 `scripts/` 里，在沙箱执行**——不是让大模型算数字
- **数据获取放在 MCP 工具**——Skill 脚本调 MCP 工具拿数据，再计算
- 这是分工：Skill 管"怎么做"，MCP 管"取什么数据"，Agent 管"决定做什么"

## 六、src/api_view/ —— Web 接口层

```
src/api_view/
├── __init__.py
├── web_main.py             ⬜ FastAPI 应用入口（步骤8）
├── web_config.py           ⬜ Web 配置
├── agent_loader.py         ⬜ Agent 懒加载
└── api/
    ├── chat.py             ⬜ 对话接口（SSE 流式，步骤8）
    ├── history.py          ⬜ 历史记录接口
    ├── portfolio_api.py    ⬜ 持仓管理接口
    └── subscribe_api.py    ⬜ 异动订阅接口
```

**这层的核心理解**：
- 对外暴露 HTTP 接口，前端通过它和 Agent 对话
- `chat.py` 用 SSE（Server-Sent Events）流式返回，用户能实时看到 Agent 的思考过程

## 七、三层架构怎么协作（一个例子）

用户问"我的持仓风险大吗"，数据怎么流：

```
用户 → api_view/chat.py（接收请求）
     → agent/main_agent.py（主 Agent 理解意图）
     → 委派 subagents/risk_analyst（风险子 Agent）
     → 加载 skills/finance/risk-analysis/SKILL.md（操作手册）
     → 子 Agent 调 mcp_server 的 portfolio_get / market_data_quote（取数据）
     → 在沙箱执行 skills/.../scripts/var_calculator.py（算 VaR）
     → 子 Agent 用 LLM 把数字翻译成人话（解读）
     → 返回主 Agent → 合成回复 → SSE 流式返回用户
```

**记住这个流向，整个项目的协作逻辑就清楚了。**

## 八、依赖关系（谁依赖谁）

```
api_view  →  agent（主 Agent）
              ├→ config / schema（地基，所有人依赖）
              ├→ subagents → skills（方法论）
              ├→ tools → mcp_server（数据）
              └→ middlewares（横切处理）
```

底层（config/schema）最稳定，越往上越业务化。改地基要谨慎，改业务层影响小。
